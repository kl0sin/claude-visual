# v0.3.1 — Bug Fixes & UX Improvements

## Bug Fixes

### macOS App Stuck on "INITIALIZING NEURAL LINK..."
The Tauri v2 desktop build failed to detect it was running inside the Tauri runtime. The app checked for `window.__TAURI__` (Tauri v1) but Tauri v2 sets `window.__TAURI_INTERNALS__` instead. As a result, the WebSocket attempted to connect to the wrong URL, which WebKit blocked via CSP, throwing a synchronous error that unmounted the entire React tree — leaving users with a permanently blank loading screen.

**Fixed:** `isInTauri()` now checks both globals. The WebSocket constructor is wrapped in a try/catch as an additional safeguard. A top-level error boundary has been added so any future fatal render errors show a recoverable error screen instead of the loading indicator.

### Agents Remaining "Active" After Completion
When a `SubagentStop` hook event arrived without a matching `agent_id`, the agent was never marked as completed and stayed in the active list indefinitely.

**Fixed:** Added fallback matching by session ID and agent type, so agents correctly transition to "completed" even when `agent_id` is absent from the hook payload.

### Agent Name Showing "Session" Instead of Actual Type
A synthetic `SubagentStart` event emitted on every `SessionStart` (used internally to track the main session lifecycle) was appearing in the Agent Processes panel with the label "Session", obscuring real subagents.

**Fixed:** Internal session-marker agents (`type: "session"`) are now excluded from the Agent Processes panel. Only real spawned subagents (e.g. `Explore`, `Plan`, `general-purpose`) are displayed.

## UX Improvements

### Running Tools Banner Reworked
The previous "TOOL STALLED — check your terminal" alert triggered for any tool running longer than 15 seconds — including normal long-running commands like builds or test runs that require no user interaction. This caused repeated false alarms and confusion when the banner stayed visible even after the user had already acted.

**Changed:** Replaced with a single neutral `◈ RUNNING — {tool}` informational banner that appears after 5 seconds for any pending tool. The banner makes no assumption about whether user input is needed. A dismiss button (×) lets users close it manually at any time; the banner auto-clears when the tool completes.

### Dismiss Individual Sessions from Live View
Ended sessions previously accumulated in the Live view until the global "Purge" button was used, which cleared all data.

**Added:** A × button on each ended session tab lets users remove individual sessions. The session and all its events, agents, and token data are deleted from the database, and the change is broadcast to all connected clients immediately.

### Session Tab Cleaned Up
- Start time removed from the tab (still available on hover in the tooltip)
- Session ID no longer shown in the label — full ID and working directory path are shown in the tooltip
- Duration and time values are now more legible (higher contrast, `10px` font)
- Tooltip only appears when hovering over the session name, not the entire tab
- Replay (▶) and Dismiss (×) buttons now show their own tooltips instead of the session statistics tooltip

### Tooltip Viewport Clamping
The tooltip overlay previously rendered at the raw cursor position, causing it to clip or disappear beyond the browser window edge.

**Fixed:** The tooltip now measures its rendered dimensions after each update and computes a clamped position that keeps it fully within the visible viewport. When there is insufficient space above the cursor, the tooltip flips below it automatically.
