## Bug Fixes

- **Silent hook errors when Claude Visual is not running**: Hooks no longer show error output when the Claude Visual server is not active, preventing noisy failures during normal Claude Code usage.

## Improvements

- **Better Session ID display**: Improved the way Session ID is shown in the UI for easier identification.
- **Hook installation banner now mentions `jq` requirement**: The installation banner clearly informs users that `jq` is a required dependency.

## Internal / Maintenance

- **Added missing hook event types**: Hook definitions now include all supported Claude Code event types.
- **Synchronized version numbers**: All version references across the project (server API, Tauri config, Cargo.toml, frontend, landing page) are now kept in sync during releases.
- **Added `version` field to `package.json`**: Previously missing, now included for proper package versioning.
- **New `/release` skill**: Added a Claude Code skill to automate version bumping, release notes generation, and tagging.
