## New Features

- **Agent metadata badges** (`src/components/AgentTimeline.tsx`, `src/index.css`): Agent cards now display contextual badges computed from new hook event types. A `computeAgentMeta()` function scans session events for `WorktreeCreate` and `PreCompact`/`PostCompact` to derive per-agent metadata. Two badge types are rendered: worktree isolation indicator (purple) and context compaction status (orange, with pulse animation while active on live agents, `Nx compact` count on completed ones). Badges appear on both active and completed agent cards between the description and tool action list.

- **Docker & container usage documentation** (`README.md`): New section covering how to use Claude Visual when Claude Code runs inside Docker containers. Includes specific setup instructions for [cleat](https://github.com/cleatdev/cleat) (editing `CLAUDE_ENV` array and `docker run` in `bin/cleat`), Docker Compose (`environment:` block), and plain `docker run -e`. Documents the `host.docker.internal` DNS name and the Linux-specific `--add-host` workaround.

## Improvements

- **Hook timeout protection** (`hooks/claude-hooks.json`): Added `--connect-timeout 1` to all 25 curl invocations in hook commands. Previously, curl used the system default connection timeout (~120s), which caused Claude Code hooks to hang and eventually error when the Claude Visual server was not running. Now curl receives "connection refused" almost instantly on localhost, and the 1s timeout acts as a safety net for unreachable hosts. Combined with the existing `; exit 0` suffix, hooks now silently succeed regardless of server state.

- **Remote server support in hooks** (`hooks/claude-hooks.json`): Replaced hardcoded `http://localhost:3200` with `${CLAUDE_VISUAL_URL:-http://localhost:3200}` in all hook curl commands. This allows hooks running inside Docker containers or remote sandboxes to reach the Claude Visual server on the host via the `CLAUDE_VISUAL_URL` environment variable, while preserving localhost as the default for local usage.

- **Alert banner shows only for interactive events** (`src/components/Header.tsx`, `src/index.css`): Removed the slow-running tool banner that displayed "RUNNING — Bash" after 5 seconds for any pending tool. This banner was purely informational and caused noise for tools like Bash that legitimately run for extended periods without needing user input. The terminal action banner (for `PermissionRequest` and `Elicitation` events) is preserved — these are the only events that require user interaction in the terminal.

- **Project name in terminal action banner** (`src/components/Header.tsx`, `src/index.css`): The "PERMISSION REQUIRED — CHECK TERMINAL" and "INPUT REQUIRED — CHECK TERMINAL" banners now display the project directory name (e.g., `[my-project]`) extracted from the session's `cwd` field. This helps users identify which terminal to check when multiple Claude Code sessions are running simultaneously. The `sessions` prop was added to `Header` to enable session lookup by the pending event's `sessionId`.

- **Complete hook event types in CLAUDE.md** (`CLAUDE.md`): Updated the Hook Event Types table from 11 to 25 entries, covering all event types from the official Claude Code hooks documentation. Newly documented events include `InstructionsLoaded`, `PermissionRequest`, `TaskCreated`, `StopFailure`, `TeammateIdle`, `ConfigChange`, `CwdChanged`, `FileChanged`, `PreCompact`, `PostCompact`, `Elicitation`, `ElicitationResult`, `WorktreeCreate`, and `WorktreeRemove`.

## Internal / Maintenance

- **GitHub Release naming** (`.github/workflows/release.yml`): Changed `releaseName` from `'Claude Visual v0.4.4'` to `'v0.4.4'` for cleaner release titles.

- **Automated release notes in GitHub Releases** (`.github/workflows/release.yml`): Added a "Read release notes" step that reads `releases/v<VERSION>.md` and passes the content as `releaseBody` to the `tauri-apps/tauri-action`. GitHub Release drafts now automatically include the release notes content instead of being empty.

- **Technical release notes format** (`.claude/commands/release.md`): Updated the `/release` command to generate more detailed, technical release notes. The command now reads actual diffs (`git show`) for each commit and requires descriptions that include file paths, APIs used, and system impact. Added examples of good vs bad descriptions.

- **Updated add-hook command template** (`.claude/commands/add-hook.md`): Synchronized the hook command template with the current pattern — added `--connect-timeout 1`, `${CLAUDE_VISUAL_URL:-http://localhost:3200}`, and `; exit 0` to the example snippet.

- **Commands tracked in git** (`.gitignore`): Added `!.claude/commands/` exception to `.gitignore` so that Claude Code command definitions are version-controlled while other `.claude/` files (settings, cache) remain ignored.

- **New environment variables documented** (`README.md`): Added `CLAUDE_VISUAL_URL`, `CLAUDE_VISUAL_TOKEN`, and `MAX_EVENTS` to the Configuration table — these existed in the codebase but were not previously documented in the README.

- **Extended agent and tool color maps** (`src/components/AgentTimeline.tsx`): Added color entries for `statusline-setup` agent type, and `Agent`, `NotebookEdit` tools to improve visual differentiation in the agent timeline.
