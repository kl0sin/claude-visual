# v0.5.0

## New Features

- **`PermissionDenied` hook event** (`hooks/claude-hooks.json`, `server/events.ts`, `src/types.ts`, `src/components/Header.tsx`, `src/hooks/useNotifications.ts`, `src/components/SettingsPage.tsx`): Added support for the new `PermissionDenied` event type introduced in Claude Code v2.1.89. This event fires when the auto mode classifier denies a tool invocation. Wired through the full stack: hook definition with pass-through jq pattern, server event processing (no `isProcessing` flag — unlike `PermissionRequest`, this is a denial not a wait), frontend color (`#ff0040` / cyber-red) and icon (`⛔`), attention banner ("PERMISSION DENIED"), desktop/toast notifications with a dedicated toggle in Settings, and `TERMINAL_TRIGGERS` in `App.tsx` so the attention banner activates. Also added to `server/history.ts` inline hook definitions for the hook install check.

- **Deferred tools visualization** (`shared/types.ts`, `server/events.ts`, `src/components/EventFeed.tsx`): Added `deferred?: boolean` flag to the `PendingTool` interface. When a `PermissionDenied` event arrives for a tool that is currently in the pending tools list, the server marks it as deferred instead of removing it. The EventFeed processing banner now shows "Deferred: {tool}..." in cyber-yellow (`--color-cyber-yellow`) instead of the default "Running {tool}..." text, giving visual feedback that a tool is paused at a permission boundary rather than actively executing.

## Improvements

- **Increased stale processing threshold** (`src/components/SessionSelector.tsx`): Raised `STALE_PROCESSING_THRESHOLD` from 60 seconds to 180 seconds (3 minutes). The previous 1-minute timeout was too aggressive — Claude's extended thinking time between tool calls could exceed 60 seconds, causing sessions to incorrectly show as "interrupted" while Claude was still actively thinking.
